How I Use AI

Working With AI

Daily use, hands-on builds

My approach

I use AI daily, the same way I use SQL or a spreadsheet: as a tool that has to earn its place in the answer. I treat it as a copilot, useful for speed and for surfacing a first draft. The judgement on what is true, what is worth shipping, and what the data actually supports stays with me. The pieces below describe what I have built and why, without sharing the underlying code or any confidential material.

What I've built

A scheduled daily briefing

Set up an automated morning briefing that pulls together the day's priorities and relevant context before I sit down to work, so I start straight into acting on what matters. Small workflow, but it compounds daily.

Example output: APAC Employment Market Briefing (PDF), a sample market-intelligence briefing covering employer and jobseeker behaviour trends across Australia, Singapore, and the wider region, with sourced citations and marketing implications drawn out for a strategy audience.

Comparison tools

Built lightweight tools that lay out options, offers, or decisions side by side, structured around the actual trade-off I am weighing. Useful whenever a decision has more variables than fit comfortably in a spreadsheet.

Custom GPTs with guardrails

Designed custom GPTs and prompt guardrails to keep AI-assisted writing consistent in tone and language, and to catch likely hallucinations before they reach a reader. The guardrail is as important as the output: knowing where a model is likely to overreach and building a check for it.

Vibe-coded tools

Built working tools through AI-assisted coding without a traditional software engineering background, including this portfolio site and its interactive analyses. The skill here is knowing what to ask for, reading the output critically, and testing that it actually does what it claims to.

Where I draw the line

AI output gets the same scrutiny I would apply to a junior analyst's first draft: checked against the source data, checked for overreach, and rewritten where it states something with more confidence than the evidence supports. That discipline is what makes AI useful for real work.
